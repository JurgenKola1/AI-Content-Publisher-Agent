## Environment Variables

| Variable Name | What It Is Used For | Where It Is Used |
|---------------|---------------------|------------------|
| `OPENAI_API_KEY` | Authenticates requests to the OpenAI API | `backend/generator.py` |
| `OPENAI_MODEL` | Selects the text model for article and image-plan prompts | `backend/generator.py` |
| `OPENAI_IMAGE_MODEL` | Selects the primary image generation model | `backend/generator.py` |
| `ENABLE_ARTICLE_IMAGES` | Enables or disables one image per article | `backend/config.py`, `backend/generator.py` |
| `DEVTO_API_KEY` | Authenticates requests to the Dev.to API | `backend/publisher.py`, `backend/image_hosting.py` |

Do not put real API key values in `.env.example` or documentation.

## Short Project Flow

Here is how the tools connect together:

1. **User enters a topic** in the browser (HTML form in `templates/index.html`).
2. **Flask receives the request** (`backend/routes.py`).
3. **Backend loads a prompt** from `prompts/` via `backend/prompts.py`.
4. **OpenAI API generates the article** (`backend/generator.py`).
5. **Optional: OpenAI Image API creates one PNG**, saved to `drafts/images/`.
6. **catbox.moe hosts the image** so Dev.to can use a public URL (`backend/image_hosting.py`).
7. **Article is saved locally** as Markdown in `drafts/` (`backend/publisher.py`).
8. **Dev.to API creates an online draft** with `published: false` (`backend/publisher.py`).
9. **Topic status is updated** in `data/topics.json` if using the queue (`backend/topics.py`).
10. **User reviews the result** in the browser and on the Dev.to dashboard.

## Why These Tools Were Chosen

| Choice | Reason |
|--------|--------|
| Flask | Simple to learn, minimal setup, ideal for book examples |
| JSON file storage | No database setup — readers can see data directly |
| Separate prompt files | Easy to customize writing style without code changes |
| OpenAI API | Reliable text and image generation in one platform |
| Dev.to API | Free developer blogging platform with a straightforward REST API |
| catbox.moe | Dev.to needs public image URLs; no official upload API for scripts |
| Local drafts folder | Content is preserved even if online publishing fails |
